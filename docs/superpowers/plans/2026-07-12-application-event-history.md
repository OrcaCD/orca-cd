# Application Event History Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a live, application-scoped history of manual deployments, commit synchronizations, and image updates with source/actor attribution, terminal results, useful no-change records, and bounded retention.

**Architecture:** A new `application_events` model and migration persist one row per logical operation. A focused `applicationevents` package owns creation, correlation, completion, retention, and restart recovery; existing deployment, repository-sync, and image-result flows pass explicit source metadata and share protobuf request IDs with the history row. A protected paginated endpoint feeds a TanStack Router page using SWR Infinite and existing table primitives, while SSE keeps running rows current.

**Tech Stack:** Go 1.26, Gin 1.12, GORM 1.31 Generics API, SQLite/golang-migrate, protobuf WebSocket messages, React 19, TanStack Router 1.170, TanStack Table 8, SWR 2.4, shadcn/Base UI, Paraglide.

## Global Constraints

- Keep at most the newest 1,000 events per application.
- Skip successful periodic image polls when `images_updated = false`.
- Skip scheduled repository polls when the resolved commit equals the application's recorded commit.
- Record explicit/manual/webhook/GitHub Actions operations even when they produce no change.
- A history persistence failure must be logged and must not block the underlying deployment, sync, or image operation.
- Complete asynchronous events only when both `request_id` and `application_id` match; retain the Agent/application ownership check for image results.
- Use the GORM Generics API wherever it supports the operation.
- Do not modify files under `frontend/src/components/ui`.
- Do not add a frontend test framework; verify the frontend with typecheck, lint, formatting, translation check, and production build.

```go
package applicationevents

import (
	"context"
	"fmt"
	"time"

	"github.com/OrcaCD/orca-cd/internal/hub/db"
	"github.com/OrcaCD/orca-cd/internal/hub/models"
	"github.com/OrcaCD/orca-cd/internal/hub/sse"
	"gorm.io/gorm"
)

const MaxPerApplication = 1000

type Params struct {
	ApplicationID string
	RequestID *string
	Type models.ApplicationEventType
	Source models.ApplicationEventSource
	ActorUserID *string
	ActorName *string
	CommitHash *string
	CommitMessage *string
}

func Path(applicationID string) string {
	return "/api/v1/applications/" + applicationID + "/events"
}

func Start(ctx context.Context, p Params) (*models.ApplicationEvent, error) {
	return create(ctx, p, models.ApplicationEventRunning, nil)
}

func RecordTerminal(ctx context.Context, p Params, status models.ApplicationEventStatus, errorMessage *string) (*models.ApplicationEvent, error) {
	if status == models.ApplicationEventRunning {
		return nil, fmt.Errorf("terminal event cannot have running status")
	}
	return create(ctx, p, status, errorMessage)
}

func create(ctx context.Context, p Params, status models.ApplicationEventStatus, errorMessage *string) (*models.ApplicationEvent, error) {
	event := models.ApplicationEvent{
		ApplicationId: p.ApplicationID, RequestId: p.RequestID, Type: p.Type,
		Source: p.Source, Status: status, ActorUserId: p.ActorUserID,
		ActorName: p.ActorName, CommitHash: p.CommitHash,
		CommitMessage: p.CommitMessage, ErrorMessage: errorMessage,
	}
	if status != models.ApplicationEventRunning {
		now := time.Now()
		event.CompletedAt = &now
	}
	err := db.DB.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
		if err := gorm.G[models.ApplicationEvent](tx).Create(ctx, &event); err != nil {
			return err
		}
		return tx.Exec(`DELETE FROM application_events
			WHERE application_id = ? AND id NOT IN (
				SELECT id FROM application_events WHERE application_id = ?
				ORDER BY created_at DESC, id DESC LIMIT ?
			)`, p.ApplicationID, p.ApplicationID, MaxPerApplication).Error
	})
	if err != nil {
		return nil, err
	}
	sse.PublishUpdate(Path(p.ApplicationID))
	return &event, nil
}
```

Implement `Complete` as one conditional generic update with `WHERE request_id = ? AND application_id = ? AND status = 'running'`, selecting `Status`, `ErrorMessage`, and `CompletedAt` so nil/zero values are written. Publish only when `rowsAffected == 1`. Implement `RecoverRunning` as a conditional update of every running row to failed with one shared completion time and the supplied stable message, returning the affected count.
